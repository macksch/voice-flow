## Lessons Learned
- **Prompt Isolation**: Hardcoded few-shot examples can semantically contaminate LLM context; minimal, neutral examples are safer.
- **CI/CD Permissions**: GitHub Actions `GITHUB_TOKEN` defaults to read-only; explicit `contents: write` is required for releases.
- **Electron Builder**: Using `dist --publish always` is the most reliable way to trigger releases from CI.
